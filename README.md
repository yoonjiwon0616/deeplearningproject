# deeplearningproject
딥러닝실습 프로젝트: 순부하 예측을 통한 스마트 그리드

단계 1. 데이터 전처리 및 특성 공학 (Feature Engineering)
데이터 클리닝: train.csv와 제주 데이터셋의 결측치 처리 및 이상치 제거
기상 특성 산출: 기온과 습도를 조합하여 THI(불쾌지수) 및 CDH(냉방도일) 파생 변수 코드 구현
시간 주기성 인코딩: hour와 month 변수를 단순 숫자가 아닌 \sin, \cos 함수로 변환하여 연속성 부여 
데이터 표준화: MLP 모델의 안정적 학습을 위해 StandardScaler를 활용하여 변수들의 스케일 매칭 단계 

단계 2. 태양광 발전량 추정 엔진 구축 (2단계 계층적 MLP: solar_power_forecast MLP모델)
test.csv에 일사량 데이터가 없는 한계를 전이 학습과 구조적 모델링으로 돌파하는 핵심 단계
Step 1. 제주 데이터 선학습 (Pre-training)
MLP A (일사량 추정기): [기온, 습도, 풍속 등] -> [일사량] 관계 학습
MLP B (발전량 변환기): [일사량] -> [태양광 발전량] 물리적 변환 관계 학습
Step 2. 타겟(Train) 데이터 파인튜닝 (Fine-tuning)
제주도와 타겟 지역의 기후 편차를 줄이기 위해, train.csv 데이터를 사용하여 MLP A 모델만 국소적으로 파인튜닝. (이때 MLP B의 가중치는 고정)
Step 3. 테스트 데이터 추론 및 모델 저장일사량이 없는 test.csv 기상 데이터를 넣어 [MLP A -> MLP B] 체인을 통해 최종 '예상 태양광 발전량' 도출.통합 모델 객체를 solar_power_forecast.pth 파일로 저장.

단계 3. 건물별 전력 수요 예측 엔진 구축 (TimesFM)
구글의 파운데이션 모델을 활용하여 안정적인 수요 예측 모델을 만듦
데이터 패칭 (Input Patching): 과거 512시간(약 3주)의 전력 수요 시계열 데이터와 앞서 생성한 외부 변수(Covariates)를 묶어 TimesFM 입력 구조 생성
비대칭 가중 손실함수(Asymmetric Loss) 구현: 전력 부족(과소 예측) 상황에 더 큰 페널티를 주는 커스텀 Loss 함수 정의
범용 수요 예측 모델 학습 및 저장: 전체 건물 데이터를 통합하여 전역 학습을 진행하고 demand_forecast_model.pth 파일로 저장

단계 4. 순부하 산출 및 시각화 대시보드 제작 (Streamlit+FastAPI)
예측된 두 데이터를 결합하여 실무적인 인사이트를 제공하는 웹 서비스를 만듭니다.
순부하(Net Load) 연산: [예측된 전력 수요] - [예측된 태양광 발전량] 수식을 통해 최종 순부하 데이터 산출 (음수 값은 0으로 보정).[ ] 
성과 지표 검증:통계적 지표: sMAPE 산출.안정성 지표: 혼동 행렬 관점에서의 2형 오류율(Under-prediction Rate) 분석을 통해 모델이 안전 마진을 잘 확보했는지 검증.[ ] 

Streamlit 대시보드 UI 구현:사용자가 월, 시간, 건물을 선택하면 실시간으로 전력 수요, 태양광 발전량, 순부하 곡선이 겹쳐서 나오는 그래프 구현.
덕 커브 현상 경고 알림 등 실무 시나리오 기능 추가.
